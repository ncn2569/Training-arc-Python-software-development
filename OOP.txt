2.Tính thừa kế
cách để tăng độ linh động của class defintion khi lớp con có thể sử dụng lại thuộc tính hoặc phương thức của lớp cha
để kế thừa và đồng thời có thể tự mở rộng thêm thuộc tính hoặc phương thức của mình.

-> keyword là tính tái sử dụng và mối quan hệ is-a

class Animal:
    def __init__(self,legs,eyes):
        self.legs=legs
        self.eyes=eyes
    def mate(self):
        print("mating")
class Dog(Animal):
    def __init__(self,legs,eyes,tail):
        super().__init__(legs,eyes) -> gọi hàm khởi tạo của lớp cha
        self.tail=tail
    def sniff(self):
        print("sniffing")
    def mate(self)
        super().mate()
        print("mate từ tầng của loài chó")

3.Tính đa hình
biểu diễn 1 interface cho nhiều loại kiểu dữ liệu khác nhau.

Ví dụ: int, float, double là các kiểu dữ liệu khác nhau nhưng vẫn cộng trừ nhân chia được với nhau

trong OOP, bằng cách để cho class chịu trách nhiệm cho code và data của nó thì tính đa hình có thể đạt được, 
mà khi được gọi thì nó sẽ xử lý tương ứng theo bất kỳ objects nào.
nghĩa là các class tương tác với 1 hàm theo những cách khác nhau 

ghi đè phương thức -> method overriding
nạp chồng phương thức -> method overloading
ad-hoc polymorphism / duck typing 
operator overloading

class DiemSo:
    def __init__(self, diem):
        self.diem = diem

    # Định nghĩa lại toán tử cộng (+) bằng magic method __add__
    def __add__(self, other):
        return DiemSo(self.diem + other.diem)

# --- CHẠY ---
toan = DiemSo(8)
van = DiemSo(7)

# Bản chất phép cộng dưới đây là: toan.__add__(van)
tong_diem = toan + van 

print(tong_diem.diem) # Output: 15

class A:

b=B()

4.Tính trừu tượng
Giấu đi sự phức tạp của các hàm, chỉ đưa ra interface -> chỉ cần quan tâm interface và input output của nó
không cần quan tâm cách nó được hiện thực-> giảm sự phức tạp và cho phép thiết kế hiệu quả hơn.
Chuẩn thiết kế chung cho các class không quan tâm cách hiện thực của tụi function chỉ cần biết là phải có function đó là được,
giảm thiểu đi sự phức tạp, chỉ cần làm việc với cách dùng không phải với cách hiện thực của hàm.

có thể hiện thực 2 cách

from abc import ABC, abstractmethod #cách để python tạo abstract class và abstract method (ABD: abstract base class)

class computer(ABC):
    @abstractmethod
    def cpu(self):
        pass
    -> mọi class con kế thừa computer đều phải overwrite hàm này
class HP(computer):
    def cpu(self):
        print("intel")
class DELL(computer):
    def cpu(self):
        print("amd")
